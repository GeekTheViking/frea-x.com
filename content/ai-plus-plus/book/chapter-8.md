# Chapter 8 — Getting Started

## Your first AI++ session

This chapter is different from the ones before it.

Less philosophy. More practice.

The previous chapters described what AI++ is and why it matters.
This one tells you how to begin — with what you already have.

---

## What you actually need

Not a data center.
Not a GPU cluster.
Not a subscription to five different services.

You need three things:

**1. A working method**
The habit of treating every AI interaction as a session in an ongoing collaboration — not a one-off query.

**2. A memory system**
Some way of passing context from one session to the next. This can be as simple as a text file. It can grow into structured notes, linked documents, session summaries. The tool matters less than the habit.

**3. An AI that can act**
Not just answer. Act. Read files. Write things. Run commands. If your AI cannot do this — you have a search engine, not a collaborator.

---

## The First Session

Start here.

Open a session with whatever AI you have access to.
Tell it three things:

```
1. Who you are and what you are working on.
2. What decisions you have already made.
3. What you want to accomplish in this session.
```

At the end of the session, do one thing before you close it:

```
Ask the AI to summarise:
- What was decided
- What was built
- What comes next
```

Save that summary somewhere.
That is your memory system. Version 1.

---

## The Second Session

Start the second session by giving the AI the summary from the first.

Watch what changes.

You do not explain your project again from the beginning.
You do not re-establish context.
You arrive where you left off.

That is AI++. In its simplest form.
Before any special tools.
Before any infrastructure.
Just a habit.

---

## Growing the Practice

Over time, the memory system grows.

What starts as a text file becomes structured notes.
Notes become a personal wiki.
A personal wiki becomes a system that the AI can read and write directly.

What starts as a single AI becomes a family.
KANDA for daily questions.
LO for the work that stays close to home.
The PROSTEIN Professors for the deep questions.

What starts as a working method becomes an infrastructure.
Local hardware running open models.
Sessions that begin with full context.
Decisions that accumulate into something that would have taken years to build without this.

---

## The Local Stack

If you want to go further:

**Hardware minimum:** A modern laptop or a Raspberry Pi 5 with AI HAT+.
**Model minimum:** A 1-3B GGUF model running via Ollama or llama.cpp.
**Memory minimum:** A plain text file, a Markdown wiki, or Trilium Notes.
**Agent minimum:** Claude Code, or any AI that can read and write files and run commands.

This is not a shopping list.
It is a starting point.

Every item can be upgraded. Every item can be replaced.
The stack is not the point. The practice is.

---

## A Note on Starting Imperfectly

You will not do this perfectly the first time.

The memory system will have gaps.
The AI will misunderstand context.
The local hardware will be slower than you expected.

This is correct.

C++ programmers wrote terrible C++ in 1985.
They got better.
The language got better.
The tools got better.
The practice refined itself over time.

AI++ is in 1985.

Start now.
Start imperfectly.
Contribute to making it better.
