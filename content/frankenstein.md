# FRANKENSTEIN AI

> *It was ugly. It was assembled from parts.*
> *But it lived.*

## What is FRANKENSTEIN?

FRANKENSTEIN is not a product.
It is not a startup. It is not a pitch deck.

FRANKENSTEIN is a CyberSpace campus for hardware.
Everyone is welcome. Bring what you have.

It is infrastructure — assembled from what others left behind.

Surplus hardware from datacenters going dark.
CPU cycles donated while Asia sleeps and Europe wakes.
Mini PCs rescued from corporate recycling bins.
Community rack space. Volunteer uptime. Shared belief.

It runs on what the giants threw away.
And it runs.

---

## Why FRANKENSTEIN?

Because Frankenstein did not create a monster.

He created *life* — from parts that were dead, discarded, forgotten.
The horror was never the creation.
The horror was that the world did not know what to do with it.

We do.

---

## The Hardware Philosophy

```
If it computes — it counts.
If it has cycles to spare — we'll take them.
If you're done with it — we're just getting started.
```

No vendor lock-in.
No minimum specs.
No proprietary stack.

ARM, x86, GPU clusters, single-board computers.
When the big players consolidate and downsize —
we show up with a truck.

---

## The Network

FRANKENSTEIN is distributed by design.

- Nodes run where power is cheap and idle
- EU compute runs while US sleeps
- US compute runs while EU sleeps
- The grid never fully stops
- No single point of failure
- No single point of control

This is not a cloud.
This is a **mesh of committed humans.**

---

## Who builds FRANKENSTEIN?

Hackers. Students. Sysadmins who believe in something.
Universities with spare rack space.
Hackspaces with a server in the corner.
Companies whose hardware retires before it dies.

If you want to understand it — take it apart.
If you want to improve it — submit a patch.
If you want to run a node — here's the docs.

**No gatekeeping. No bullshit.**

---

## The Rules

```
1. Open source. Always.
2. No surveillance. Ever.
3. EU data stays in EU.
4. Hardware donations welcome. Control donations are not.
5. If it helps someone — ship it.
```

---

## The Output

From this assembled, unglamorous, beautiful mess —
PROSTEIN emerges.

The monster builds the servant.
The servant helps the human.

*That was the plan all along.*

## PROSTEIN AI — System-arkitektur og filosofi (arbejdsnotater)

### Tieret AI-stack
Brugeren mødes altid af det laveste lag der kan klare opgaven:
1. Hailo-10H modul — optimerede små modeller, øjeblikkeligt svar, lavt strømforbrug
2. server01 — tager over hvis Hailo ikke magter det
3. Z640 med GPU — tunge opgaver, udvikling af PROSTEIN AI til ThePostBox.eu
4. Ekstern AI (Claude osv.) — hvis opgaven ikke kan vente og kræver massiv compute

### Energibevidst scheduling — Home Assistant integration
Tunge opgaver udskydes til billig eller overskuds-strøm.
Brugeren modtager en ærlig besked:
*"Dit resultat er klar kl. 2.24 — strømpris forventes gratis/minimal da."*

Hvis alle vil have AI kl. 18 → simple opgaver løses, tunge venter.
Eller brugeren vælger selv at gå til ekstern AI hvis det ikke kan vente.
Det er brugerens valg — ikke systemets diktat.

### kWh og CO2 — ikke tokens
Tokens er abstrakt nonsens for borgere.
kWh og CO2 er forståelige, ærlige og meningsfulde måleparametre.
PROSTEIN AI's kvittering til brugeren: *"Dit svar kostede 0.004 kWh og 1.8g CO2."*
Gennemsigtighed er ikke en feature — det er en ret.

### Ressourcefordeling og tilgængelighed
En bruger der bruger mere tid på at formulere sig — fx pga. ordblindhed —
fortjener ikke færre ressourcer. De fortjener måske flere.
PROSTEIN AI straffer ikke dem der har sværest ved at kommunicere.
Ressourcer fordeles efter behov, ikke efter hastighed.

### Om dobbeltmoralen
At bruge eksisterende AI-værktøjer mens man bygger alternativet er ikke hyklerisk.
Det er sådan alle bevægelser fungerer.
Andelsmejerierne brugte eksisterende landeveje mens de byggede deres eget net.
Kompassretningen tæller — ikke startpunktet.
